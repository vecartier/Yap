  <behavior>
    SlackFormatter.format(_:) behavior:
    - Given a Summary with meetingTitle "Standup", date 2026-03-21, all sections populated →
      output contains "*Meeting: Standup — " (header line with bold)
    - Given a Summary with non-empty decisions ["Shipped feature X"] →
      output contains "• Shipped feature X"
    - Given a Summary with empty decisions [] →
      output contains "• _None recorded_" under Key Decisions (section still present)
    - Given a valid Summary → output contains all five section headers:
      "*Key Decisions*", "*Action Items*", "*Discussion Points*", "*Open Questions*"
    - Given a Summary with multiple action items → each appears on its own "• " bullet line

    transcriptRows(for:) behavior (free function, signature: [(SessionRecord, Bool)]):
    - Given [] → returns []
    - Given [record at T=0] → returns [(record, true)] — first record always shows timestamp
    - Given [record at T=0, record at T=60] → returns [(r0, true), (r60, false)] — 60s < 120s threshold
    - Given [record at T=0, record at T=120] → returns [(r0, true), (r120, true)] — exactly 120s triggers new marker
    - Given [record at T=0, record at T=119] → returns [(r0, true), (r119, false)] — 119s does not trigger
    - Given [r0=T0, r1=T60, r2=T130] → returns [(r0,true), (r1,false), (r2,true)] — r2 is 130s after last marker (r0)
  </behavior>
